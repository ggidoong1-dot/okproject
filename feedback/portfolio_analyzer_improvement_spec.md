## 1. 현행 분석기 진단

현재 분석기의 출력 구조(`portfolio_analysis` JSON)를 기반으로 평가한 결과, 아래 6가지 구조적 문제가 식별되었습니다.

| # | 문제 영역 | 설명 | 영향도 |
|---|----------|------|--------|
| 1 | **S-RIM 밸류에이션** | ROE/장부가치 기반 모델이라 성장주에서 구조적으로 비현실적 결과 출력 (TSLA S-RIM $17~$21 vs 현재가 $397) | 🔴 치명적 |
| 2 | **스코어카드 가중치 고정** | 밸류에이션 25% 가중치가 성장주에서 항상 최하점 → 종합점수 구조적 왜곡 | 🔴 높음 |
| 3 | **수급 데이터 부재 처리** | 해외 주식 수급 미제공 시 50점(중립) 대체 → 점수 안정성 착시 | 🟡 중간 |
| 4 | **기술적 지표 충돌 미해석** | 역배열(MA) + 골든크로스(MACD) + 매집(OBV) 동시 발생 시 해석 부재, "주의" 한 줄 요약으로 정보 손실 | 🔴 높음 |
| 5 | **뉴스 감성 분석 평면적** | 124건 중 71% 중립, 긍/부정 19:17 → 실질적 신호 부재. 시간/소스 가중치 없음 | 🟡 중간 |
| 6 | **벤치마크 부재** | 종목 절대 평가만 존재, 지수 대비 상대 수익률 비교 없음 | 🟡 중간 |

> ⚠️ **핵심**: 이 분석기는 한국 시장 저PBR 가치주에 최적화되어 있으며, 미국 성장주 / 횡보 구간 / 해외 주식에서 점수 신뢰도가 구조적으로 하락합니다.

## 2. Phase 1 — 핵심 개선 (스코어카드 재설계)

Phase 1은 기존 코드베이스에서 가장 적은 변경으로 가장 큰 품질 개선을 달성하는 것을 목표로 합니다.

### 2-1. 종목 유형 자동 분류

종목을 **성장주 / 가치주 / 배당주** 3가지로 자동 분류하고, 각 유형에 맞는 분석 프리셋을 적용합니다.

**분류 룰 (Yahoo Finance 데이터 기반):**

| 유형 | 분류 조건 | 예시 종목 |
|------|----------|----------|
| 성장주 | PER ≥ 30 AND 배당수익률 < 1% AND 매출성장률 ≥ 15% | TSLA, NVDA, PLTR |
| 가치주 | PBR < 1.5 AND PER < 15 AND 배당수익률 < 3% | 삼성전자, POSCO, KB금융 |
| 배당주 | 배당수익률 ≥ 3% AND 배당성장 연속 3년 이상 | KO, JNJ, 하나금융 |
| 기타 | 위 조건 미충족 → 가치주 프리셋 기본 적용 | - |

**구현 참고:**
- `yfinance`의 `info['trailingPE']`, `info['priceToBook']`, `info['dividendYield']`, `info['revenueGrowth']` 필드로 분류
- 경계값은 조정 가능하도록 config로 분리

### 2-2. 유형별 가중치 프리셋

분류된 종목 유형에 따라 스코어카드 가중치를 자동 전환합니다.

| 항목 | 현행 (고정) | 성장주 | 가치주 | 배당주 |
|------|-----------|--------|--------|--------|
| 기술적 분석 | 25% | 25% | 20% | 15% |
| **밸류에이션** | **25%** | **10%** ← 핵심 변경 | 30% | 25% |
| 수급 | 15% | 15% | 15% | 10% |
| 리스크 | 10% | 15% | 10% | 10% |
| 전략 시그널 | 10% | 15% | 10% | 10% |
| 뉴스 감성 | 15% | 20% | 15% | 10% |
| 배당 품질 (신규) | - | - | - | **20%** |

> 성장주: 밸류에이션 가중치 25% → 10%로 낮추고, 뉴스/전략시그널/리스크를 올려 성장주 특성을 반영. 배당주는 배당품질 항목을 신설하여 20% 배정.

### 2-3. 밸류에이션 모델 교체

S-RIM을 성장주에 적용하는 것은 구조적으로 부적합합니다. 유형별로 다른 밸류에이션 모델을 적용합니다.

| 유형 | 적용 모델 | 데이터 소스 |
|------|----------|------------|
| 성장주 | **PEG Ratio + PSR + EV/EBITDA** → 섹터 평균 대비 상대 비교 | yfinance: `pegRatio`, `priceToSalesTrailing12Months`, `enterpriseToEbitda` + 섹터 평균 |
| 가치주 | S-RIM (현행 유지) + PER/PBR 섹터 비교 | 현행 데이터 + 섹터 평균값 |
| 배당주 | DDM(배당할인모델) + 배당성장률 + Payout Ratio | yfinance: `dividendRate`, `payoutRatio`, `dividendGrowth` + 금리 데이터 |

**성장주 밸류에이션 점수 산출 로직 (예시):**

각 지표를 섹터 평균 대비 백분위로 변환 후 종합합니다:
- PEG < 섹터 평균: 가점 (성장 대비 저평가)
- PSR < 섹터 평균: 가점 (매출 대비 저평가)
- EV/EBITDA < 섹터 평균: 가점
- 세 지표 가중평균 → 0~100 점수로 정규화

> ✅ S-RIM $21 같은 비현실적 절대값 대신, "섹터 평균 대비 PEG 1.2배" 같은 상대 비교는 훨씬 유용한 정보를 제공합니다.

### 2-4. 수급 항목 동적 제거 및 가중치 재분배

해외 주식처럼 수급 데이터가 없는 경우, 50점 대체 대신 해당 항목을 제거하고 나머지 항목의 가중치를 비례 재분배합니다.

**예시: 성장주 + 해외주식 (수급 미제공)**

| 항목 | 원래 가중치 | 수급 제거 후 | 변화 |
|------|-----------|------------|------|
| 기술적 분석 | 25% | 29.4% | +4.4%p |
| 밸류에이션 | 10% | 11.8% | +1.8%p |
| 수급 | 15% | **제거** | -15%p |
| 리스크 | 15% | 17.6% | +2.6%p |
| 전략 시그널 | 15% | 17.6% | +2.6%p |
| 뉴스 감성 | 20% | 23.5% | +3.5%p |

**구현:**
```python
if supply_class == '해외 주식은 수급 데이터 미제공':
    del weights['수급']
    total = sum(weights.values())
    weights = {k: v / total for k, v in weights.items()}
```

## 3. Phase 2 — 분석 품질 개선

### 3-1. 기술적 지표 충돌 해석 로직

복수 지표가 상충할 때, 우선순위 트리로 종합 판정을 내립니다.

| 조건 조합 | 종합 판정 |
|----------|----------|
| MA 역배열 + MACD 골든크로스 + OBV 상승 | "하락 추세 내 반등 시도 구간 — 단기 매수 기회 탐색, 추세 전환 미확인" |
| MACD 골든크로스 + 거래량 감소 | "신뢰도 낮은 골든크로스 — 거래량 확인 후 진입 권장" |
| MA 정배열 + RSI > 70 + MFI > 80 | "과열 구간 — 추격 매수 자제, 분할 매도 검토" |
| ADX < 20 + 모든 지표 중립 | "방향성 부재 — 횡보 구간, 관망 권장" |
| MA 역배열 + RSI < 30 + OBV 하락 | "강한 하락 추세 — 손절 기준 엄격 관리, 신규 매수 자제" |

**구현:**
- if-elif 체인으로 조건 조합 매칭
- `overall` 필드를 한 줄 요약에서 상세 해석 문장으로 교체
- 신뢰도 표시(높음/중간/낮음) 추가 권장

### 3-2. 뉴스 감성 가중치 차별화

현행 동일 가중치 카운팅 대신, 두 가지 차원의 가중치를 적용합니다:

**a) 시간 가중치:**
- 7일 이내 기사: 가중치 1.5x
- 7~30일 기사: 가중치 1.0x
- 30일 초과 기사: 가중치 0.5x

**b) 소스 가중치:**
- 주요 경제지/통신사 (Reuters, Bloomberg, 매경, 한경): 1.5x
- 전문 분석 사이트 (Investing.com, TradingKey): 1.0x
- 블로그/커뮤니티: 0.5x

> ✅ 기사 데이터에 이미 `date`와 `source` 필드가 있으므로 추가 크롤링 없이 적용 가능.

### 3-3. 벤치마크(지수 대비) 비교 추가

종목 절대 수익률만으로는 성과를 정확히 판단할 수 없습니다. 같은 기간 벤치마크 지수 대비 상대 성과를 함께 표시합니다.

**추가 필드:**
- `benchmark_ticker`: US → `^GSPC` (S&P500), KR → `^KS11` (KOSPI)
- `benchmark_return`: 매입일~현재 동일 기간 벤치마크 수익률
- `alpha`: `profit_rate - benchmark_return`

**예시:** TSLA +41.8% / S&P500 +35.0% → 초과 수익 +6.8%p

**구현:**
```python
import yfinance as yf
benchmark = yf.download(benchmark_ticker, start=매입일, end=today)
benchmark_return = (benchmark['Close'][-1] / benchmark['Close'][0] - 1) * 100
alpha = profit_rate - benchmark_return
```

## 4. Phase 3 — 확장 기능

### 4-1. 가격 알림 연동

ATR 기반 손절/익절선을 실시간으로 모니터링하고, 도달 시 텔레그램 알림을 발송합니다.
- 텔레그램 봇 API로 알림 발송
- 모니터링 주기: 5분~15분 (yfinance 또는 실시간 API)
- 알림 조건: 손절선/익절선 도달, RSI 과매수/과매도, 급등락 등

### 4-2. 포트폴리오 레벨 분석

복수 종목 보유 시 포트폴리오 전체의 건강성을 평가합니다.
- 섹터 집중도 (HHI 지수)
- 종목 간 상관관계 매트릭스
- 포트폴리오 베타 (시장 민감도)
- 전체 가중평균 스코어

### 4-3. DCF 밸류에이션 (장기 목표)

애널리스트 컨센서스 기반 간이 DCF를 제공합니다. 정밀 DCF는 자동화 한계가 있으므로, 참고 수치로 표시하며 주의 문구를 함께 출력합니다.
- `earningsTrend` API로 EPS 추정치 취득
- 성장률 가정 3단계 (낙관/중립/비관)
- 할인율: WACC 또는 사용자 입력

## 6. 기술 참고사항

### 6-1. 데이터 소스

| 데이터 | API / 소스 | 비고 |
|--------|-----------|------|
| 기본 재무제표 | yfinance (`info` dict) | PER, PBR, ROE, 배당, 섹터, 성장률 등 |
| 섹터 평균값 | yfinance Sector/Industry 그룹별 집계 | PEG, PSR, EV/EBITDA 섹터 평균 산출용 |
| 벤치마크 지수 | `yfinance.download('^GSPC')` / `('^KS11')` | 기간 수익률 비교용 |
| EPS 컨센서스 | yfinance `earningsTrend` / `growthEstimates` | Phase 3 DCF 입력값용 |

### 6-2. JSON 출력 구조 변경사항

Phase 1~2 완료 후 `portfolio_analysis` JSON에 추가/변경되는 필드:

| 필드 | 위치 | 설명 |
|------|------|------|
| `stock_type` | 종목 루트 | `'growth'` \| `'value'` \| `'dividend'` \| `'other'` |
| `valuation_model` | `fundamental` | 적용된 모델명 + 세부 지표 |
| `sector_comparison` | `fundamental` | PEG/PSR/EV-EBITDA 섹터 평균 대비 배수 |
| `scorecard.active_weights` | `scorecard` | 실제 적용된 가중치 (수급 제거 반영) |
| `technical.interpretation` | `technical` | 충돌 해석 결과 문장 + 신뢰도 |
| `news.weighted_score` | `news` | 시간/소스 가중 적용된 감성 점수 |
| `benchmark` | 종목 루트 | `{ ticker, return, alpha }` |
