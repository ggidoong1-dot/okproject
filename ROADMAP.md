# Stock Memory - 주식 포트폴리오 분석 대시보드 로드맵

## 프로젝트 개요
Google Sheets/증권사 API에서 보유 주식 데이터를 불러와 기술적/펀더멘털/수급 분석을 자동 수행하고,
AI 질의응답까지 제공하는 Streamlit 대시보드.

## 기술 스택
- **프레임워크**: Streamlit (Python)
- **데이터 소스**: Google Sheets API (gspread), 한국투자증권 KIS OpenAPI
- **주가 데이터**: pykrx (한국 주식)
- **재무 데이터**: DART OpenAPI
- **차트**: plotly
- **AI**: Claude API (Anthropic)
- **배포**: Streamlit Cloud

## UI 설계 원칙: 3-Layer 구조
```
Layer 1 (메인)    → 종합 스코어카드 + 등급만 표시
Layer 2 (탭/확장) → 카테고리별 상세 지표
Layer 3 (AI)     → "왜 이 점수야?" 질문하면 AI가 설명
```

### 이평 분류 로직
```
정배열: 현재가 > 이평5 > 이평10 > 이평20 > 이평50 → [정배열] 강한 상승 추세
역배열: 이평50 > 이평20 > 이평10 > 이평5 > 현재가 → [역배열] 강한 하락 추세
그 외: [혼조세] 방향성 탐색 구간
```

### 거래량 분류 로직
```
현재 거래량 > 모든 평균         → 거래량 폭증 (과열/돌파 가능)
장기 > 단기 (감소 추세)         → 단기 과열 후 휴식 (관망)
단기 > 장기 (증가 추세)         → 거래량 훈조 (추세 탐색)
```

### 종합 분류 로직
```
정배열 + 거래량 증가 → [매수 고려]
역배열 + 거래량 감소 → [매도 고려]
그 외              → [관망] 확실한 신호 대기
```

### 각 지표 계산 공식
```python
# RSI (14일)
delta = close.diff()
gain = delta.where(delta > 0, 0).rolling(14).mean()
loss = (-delta.where(delta < 0, 0)).rolling(14).mean()
RS = gain / loss
RSI = 100 - (100 / (1 + RS))

# MACD
EMA12 = close.ewm(span=12).mean()
EMA26 = close.ewm(span=26).mean()
MACD = EMA12 - EMA26
Signal = MACD.ewm(span=9).mean()
Histogram = MACD - Signal

# 볼린저 밴드
SMA20 = close.rolling(20).mean()
STD20 = close.rolling(20).std()
Upper = SMA20 + 2 * STD20
Lower = SMA20 - 2 * STD20

# 스토캐스틱
Low14 = low.rolling(14).min()
High14 = high.rolling(14).max()
%K = (close - Low14) / (High14 - Low14) * 100
%D = %K.rolling(3).mean()

# ADX (14일)
+DM = high.diff().where((high.diff() > 0) & (high.diff() > -low.diff()), 0)
-DM = (-low.diff()).where((-low.diff() > 0) & (-low.diff() > high.diff()), 0)
TR = max(high-low, abs(high-prev_close), abs(low-prev_close))
+DI = 100 * EMA(+DM, 14) / EMA(TR, 14)
-DI = 100 * EMA(-DM, 14) / EMA(TR, 14)
DX = abs(+DI - -DI) / (+DI + -DI) * 100
ADX = EMA(DX, 14)

# ATR (14일)
TR = max(high-low, abs(high-prev_close), abs(low-prev_close))
ATR = TR.rolling(14).mean()

# OBV
OBV[i] = OBV[i-1] + (volume if close > prev_close else -volume if close < prev_close else 0)

# MFI (14일)
TP = (high + low + close) / 3
RawMF = TP * volume
PositiveMF = sum(RawMF where TP > prev_TP, 14)
NegativeMF = sum(RawMF where TP < prev_TP, 14)
MFR = PositiveMF / NegativeMF
MFI = 100 - (100 / (1 + MFR))
```

### S-RIM 공식
```python
# S-RIM (초과이익 모델)
# V = BPS + (ROE - 요구수익률) * BPS / 요구수익률
# 요구수익률: BBB- 회사채 수익률 (약 8~10%)
# ROE: 최근 3년 평균 ROE
# BPS: 최근 분기 자기자본 / 발행주식수

적정주가 = BPS + (ROE - required_return) * BPS / required_return
# 할인율 적용 (초과이익 지속 가정에 따라)
낙관적 = BPS + excess_earnings / required_return          # 영구 지속
중립적 = BPS + excess_earnings / (required_return + 0.1)  # 10% 감소
비관적 = BPS + excess_earnings / (required_return + 0.2)  # 20% 감소
```

### 한국형 CANSLIM 변형 (4단계에서 활용)
```
C - 당기순이익 증가율 (전년 대비)       → DART
A - 연간 순이익 3년 성장 추세          → DART
N - 신사업/신제품 (뉴스 기반)          → 뉴스 피드에서 AI 판단
S - 외국인+기관 순매수 추이            → KRX 수급 (원본 I항목 통합)
L - 업종 내 상대강도 (RSI 비교)        → 계산
M - KOSPI/KOSDAQ 시장 방향            → 지수 데이터
```

### 미너비니 추세 템플릿 8개 조건
```
1. 현재가 > 150일 이평
2. 150일 이평 > 200일 이평
3. 200일 이평 최소 1개월간 상승 추세
4. 50일 이평 > 150일 이평 > 200일 이평
5. 현재가 > 50일 이평
6. 현재가 > 52주 최저가 대비 +30% 이상
7. 현재가 > 52주 최고가 대비 -25% 이내
8. RS(상대강도) 점수 70 이상
```

### 터틀 트레이딩 규칙
```
진입: 20일 최고가 돌파 시 매수 (단기) / 55일 최고가 돌파 시 매수 (장기)
청산: 10일 최저가 이탈 시 매도 (단기) / 20일 최저가 이탈 시 매도 (장기)
포지션 사이징: 계좌의 1% 리스크 / ATR = 매수 수량
```
